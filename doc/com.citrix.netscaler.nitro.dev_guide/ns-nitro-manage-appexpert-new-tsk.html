AppExpert Application Management

Updated: 2012-03-14

An AppExpert application is a collection of configuration information that you set up for securing and optimizing traffic for a web application, such as Microsoft Outlook Web Access or Microsoft SharePoint, on a Citrix® NetScaler® appliance. For more information, see AppExpert.

This section describes the AppExpert application operations that can be performed using NITRO.

  • Export an AppExpert Application
  • Import an AppExpert Application

The following table describes the AppExpert operations. For a more detailed description, see the API reference available in the <NITRO_SDK_HOME>/doc folder.

Export an AppExpert Application

When an AppExpert application is exported, the following XML files are created:

  • Application template file. Contains the AppExpert application's configuration information, such as application units, rules, and configured policies. The template file is stored in the /nsconfig/nstemplates/applications directory of the NetScaler appliance.
  • Deployment file. Contains deployment-specific information of the AppExpert application, such as public endpoints, services, and configured variables. The deployment file is stored in the /nsconfig/nstemplates/applications/deployment_files/ directory of the NetScaler appliance.
Description Sample
To export an AppExpert application, do the following:
  1. Set the required properties for the export operation, using the corresponding property names of the application class.
  2. Export the AppExpert application by using the static export() method.

To export an AppExpert application named "MyApp1":

application myapp = new application();
myapp.set_appname("MyApp1");
myapp.set_apptemplatefilename("myapp_template");
application.export(ns_session,myapp);
Note: If the names of the files are not provided, the files are given the name of the AppExpert application that you are exporting. For example, if you export an AppExpert application named "MyApp1," the template file and the deployment file are named MyApp1.xml and MyApp1_deployment.xml, respectively.

Import an AppExpert application

You can import either the template file only or both the template file and the deployment file:

  • If you import the template file only, you can set parameters to configure your own deployment settings (such as public endpoints (VIPs), services, and variable values).
  • If you import both the files, the application's configuration information is imported from the template file and the deployment-specific information is imported from the deployment file.
Description Sample
To import an AppExpert application, do the following:
  1. Set the required properties for the import operation, using the corresponding property names of the application class.
  2. Import the AppExpert application by using the static Import() method.

To import an AppExpert application named "MyApp1":

application myapp = new application();
myapp.set_appname("MyApp1");
myapp.set_apptemplatefilename("myapp_template");
application.Import(ns_session,myapp);
Note: Before you import an AppExpert application, you must manually upload the template and/or deployment files to the appropriate directories on the NetScaler appliance.
