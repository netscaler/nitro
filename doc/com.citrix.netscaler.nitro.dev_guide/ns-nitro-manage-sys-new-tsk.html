System Management

Updated: 2012-03-14

This section describes the system level operations that can be performed on the NetScaler appliance.

  • Connect to the NetScaler Appliance
  • Enable NetScaler Features
  • Enable NetScaler Modes
  • Save Configurations
  • Clear Configurations
  • Disconnect from the NetScaler Appliance

The following table describes the system level operations. For a more detailed description of the operations, see the API reference available in the <NITRO_SDK_HOME>/doc folder.

Connect to the NetScaler Appliance

Before you perform any operation, you must authenticate and establish a session with the appliance.

Description Sample
To connect to the NetScaler appliance, do the following:
  1. Establish a connection to the appliance by using a constructor of the nitro_service class.
    Note: To access the NITRO API securely, use HTTPS as the protocol.
  2. Log in to the NetScaler appliance by using one of the login() methods provided by the nitro_service class.
Note: By default, the connection with the appliance expires after 30 minutes of inactivity. To change the session timeout period, specify the new timeout period in the login() method.

To connect to a NetScaler appliance with NSIP address 10.102.29.60 by using the using HTTP protocol:

//Specify the NetScaler appliance IP address and protocol
nitro_service ns_session = new nitro_service("10.102.29.60","http");

//Specify the login credentials
ns_session.login("admin","verysecret");
Note: You must have a user account on that appliance. The configuration operations that you can perform are limited by the administrative role assigned to your account.

Enable NetScaler Features

NetScaler features must be enabled explicitly before using them (except features that are enabled by default). The features that can be enabled depend on the NetScaler license.

Description Sample

To enable or disable features, invoke the enable_features() method or disable_features() method.

To get a list of features available or features currently enabled, invoke the get_features() method or get_enabled_features() method.

To enable the load balancing feature:

ns_session.enable_features("lb");

Enable NetScaler Modes

Modes represent NetScaler capabilities that need to be enabled in order to take effect. Modes are not related to licensing. They are capabilities that affect traffic processing, and they can be turned on or off as required.

Description Sample

To enable or disable modes, invoke the enable_modes() method or disable_modes() method.

To get a list of the modes available or modes currently enabled, invoke the get_modes() method or get_enabled_modes() method.

To enable L2 mode:

ns_session.enable_modes("L2");

Save Configurations

NetScaler configurations must be saved before disconnecting. Otherwise, the configurations do not persist.

Description Sample

To save the current configurations on the appliance, invoke the save_config() method.

To save NetScaler configurations:

ns_session.save_config();

Clear Configurations

NetScaler configurations can be cleared if you do not want the changes to persist. You can clear basic configurations, extended configurations, or all configurations.

Description Sample

To clear the current configurations from the appliance, invoke the clear_config() method, including a Boolean "true" as a parameter to indicate that you want to force clearing the configuration even if it has not been saved.

You can also specify whether you want to clear basic, extended, or full configuration, by passing a string as the second string parameter for the clear_config() method.

To clear all NetScaler configurations:

ns_session.clear_config();
Note: If you are using HTTPS to connect to the appliance, clearing the configuration also clears the SSL configuration on the appliance, which results in immediate disconnection.

Disconnect from the NetScaler Appliance

When the application does not need to interact with the appliance, you must log off the appliance.

Description Sample

To disconnect from the NetScaler appliance, invoke the logout()method.

To disconnect from a NetScaler appliance:

ns_session.logout();
Note: Make sure that you have saved the configurations before performing this operation.
