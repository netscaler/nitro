Introduction: 

This document describes NetScaler Java NITRO API, a library that enables to configure NetScaler.

Directory Structure:

After unzipping nitro-java.tgz to nitro-java directory, it will have following directory structure for programming in JAVA

       nitro-java/src:

             • This directory contains source code for the APIs.

       nitro-java/lib:  

             • gson.jar is the library used for serializing/desterilizing java objects into JSON and vice-versa.
             • nitro.jar – This is NetScaler NITRO API jar file.

       nitro-java/sample:

             • Contains samples written in java that uses NITRO API to configure/ monitor Netscaler.

       nitro-java/doc:

             • Contains documentation for java API.

Requirements:

For the examples to work, you need the JSDK 1.5.0 and later.

Samples:

         get_config.java:

         The get_config example fetches information about all configured entities and prints selected information about each one.

         set_config.java:

         The set_config example uses the NetScaler NITRO API to set and bind certain entities in the System.

         rm_config.java:

         The rm_config example removes all configuration that was created by set_config.java

         stats.java

         The stats example demonstrates usage of statistical api.

         apptemplate_import.java:

         The apptemplate_import example imports the application from apptemplate file.

         apptemplate_export.java:

         The apptemplate_export example exports the application to apptemplate file.

Usage:

          • Modify the path of JDK_HOME in build.bat and run.bat
          • Compiling sample Client application : build.bat
             The Client application is compiled with API jar file and generates class files
          • Finally, execute to configure the NetScaler box remotely using  run.bat.
             run <NSIP> <username> <password>

Example 1:

step1: Creating nitro_service object:

          nitro_service client = new nitro_service(ip);

step2: Login to NetScaler:

          client.login(username,password);

step3: Get all the load balancing virtual servers configured on NetScaler:

         lbvserver[] result = lbvserver.get(client);

step4: logout of the NetScaler:

          client.logout();

Example 2:

step1: Creating nitro_service object:

          nitro_service client = new nitro_service(ip);

step2: Login to NetScaler:

          client.login(username,password);

step3: Add a load balancing virtual server to NetScaler:

         lbvserver obj = new lbvserver();

         obj.set_name("lb_vip");

         obj.set_servicetype("HTTP");

         lbvserver.add(client, obj);

step4: logout of the NetScaler:

          client.logout();

Example 3:

step1: Creating nitro_service object:

          nitro_service client = new nitro_service(ip);

step2: Login to NetScaler:

          client.login(username,password);

step3: modify/update a load balancing virtual server on NetScaler:

         lbvserver obj = new lbvserver();

         obj.set_name("lb_vip");

         obj.set_lbmethod("ROUNDROBIN");

         lbvserver.update(client, obj);

step4: logout of the NetScaler:

          client.logout();

Example 4:

step1: Creating nitro_service object:

          nitro_service client = new nitro_service(ip);

step2: Login to NetScaler:

          client.login(username,password);

step3: remove a load balancing virtual server on NetScaler:

          lbvserver.delete(client, "lb_vip");

sstep4: logout of the NetScaler:

          client.logout();
